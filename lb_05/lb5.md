## Задание 1 

**Условие:**
Найти дилеров (`dealerships`) в штате `TX`.

**Запрос:**

```sql
EXPLAIN ANALYZE
SELECT *
FROM dealerships
WHERE state = 'TX';
```

**План выполнения (Seq Scan):** 
<img width="965" height="352" alt="image" src="https://github.com/user-attachments/assets/397c081c-4cd0-4e14-8ed1-5ed442b45b02" />

**Результат анализа:**

* Тип сканирования: **Seq Scan (последовательное)**
* Стоимость: `cost=0.00..1.25`
* Время выполнения: **Execution Time: 5.590 ms**
* Отфильтровано строк: 17
* Найдено строк: 3

**Вывод:**
Используется полный перебор таблицы, так как индекс отсутствует. Это приводит к лишним затратам ресурсов и увеличению времени выполнения запроса.

## Задание 2 (Локально)

**Условие:**
Оптимизировать поиск дилеров по штату с помощью индекса.

**Создание индекса:**

```sql
CREATE INDEX idx_dealerships_state
ON dealerships(state);
```
<img width="467" height="106" alt="image" src="https://github.com/user-attachments/assets/3f0b2c4a-25b6-4f6d-a93e-82984beef5c3" />

**Повторный запрос:**

```sql
EXPLAIN ANALYZE
SELECT *
FROM dealerships
WHERE state = 'TX';
```

**План выполнения после оптимизации:**

<img width="965" height="352" alt="image" src="https://github.com/user-attachments/assets/ad850bbe-60e6-45bf-9917-373683073ac3" />

**Результат анализа:**

* Тип сканирования: **Seq Scan** (индекс не был использован)
* Стоимость: `cost=0.00..1.25`
* Время выполнения: **Execution Time: 0.046 ms**

---

### Важное замечание

Несмотря на создание индекса, PostgreSQL **не использовал его**, так как:
* таблица очень маленькая
* дешевле сделать Seq Scan, чем использовать индекс

---

### Сравнение производительности

| Параметр         | До индекса | После индекса |
| ---------------- | ---------- | ------------- |
| Тип сканирования | Seq Scan   | Seq Scan      |
| Время выполнения | 5.590 ms   | 0.046 ms      |

**Вывод:**
Время выполнения уменьшилось с **5.590 ms до 0.046 ms**, однако это связано не с индексом, а с кэшированием данных (Buffers: shared hit).
Для маленьких таблиц использование индекса нецелесообразно.

## Задание 3 (Локально)

**Условие:**
Оптимизировать поиск клиентов по email (уникальный поиск).

**Запрос:**

```sql
SELECT *
FROM customers
WHERE email = 'test@example.com';
```

---

**Создание индекса:**

```sql
CREATE UNIQUE INDEX idx_customers_email
ON customers(email);
```

---

**Проверка:**

```sql
EXPLAIN ANALYZE
SELECT *
FROM customers
WHERE email = 'test@example.com';
```

---

**Ожидаемый результат:**

* Тип сканирования: **Index Scan**
* Значительное ускорение выполнения запроса
<img width="1239" height="360" alt="image" src="https://github.com/user-attachments/assets/157861c2-0fee-4d72-9571-5229fdebf33b" />

---

**Вывод:**
Для поиска по уникальному полю (`email`) использование **UNIQUE индекса** является наиболее эффективным. Это обеспечивает быстрый доступ к данным и минимальное время выполнения запроса.
