## 3. Индивидуальные задания

### Задание 1

**Условие:**
Пронумеровать уникальных клиентов в таблице `sales` по сумме их покупок (от большей к меньшей).

**Решение:**

```sql
SELECT 
    customer_id,
    SUM(sales_amount) as total_sales,
    ROW_NUMBER() OVER (ORDER BY SUM(sales_amount) DESC) as row_number,
    RANK() OVER (ORDER BY SUM(sales_amount) DESC) as rank,
    DENSE_RANK() OVER (ORDER BY SUM(sales_amount) DESC) as dense_rank
FROM sales
GROUP BY customer_id
ORDER BY total_sales DESC;
```

**Пояснение:**

* `SUM(sales_amount)` — вычисляет общую сумму покупок клиента.
* `ROW_NUMBER()` — присваивает уникальный номер каждой строке.
* `RANK()` — присваивает одинаковый ранг при равных значениях.
* `DENSE_RANK()` — ранжирует без пропусков.

### Задание 2

**Условие:**
Сравнить дату найма сотрудника с датой найма следующего принятого сотрудника в том же дилерском центре.

**Решение:**

```sql
SELECT 
    salesperson_id,
    dealership_id,
    first_name,
    last_name,
    hire_date,
    LEAD(hire_date) OVER (PARTITION BY dealership_id ORDER BY hire_date) as next_hire_date,
    LEAD(first_name) OVER (PARTITION BY dealership_id ORDER BY hire_date) as next_employee_name
FROM salespeople
WHERE termination_date IS NULL
ORDER BY dealership_id, hire_date;
```

**Пояснение:**

* `PARTITION BY dealership_id` — разбивает сотрудников по дилерским центрам.
* `ORDER BY hire_date` — сортирует внутри центра по дате найма.
* `LEAD()` — получает значение следующей строки (следующего сотрудника).

### Задание 3

**Условие:**
Рассчитать нарастающий итог продаж (`sales_amount`) отдельно для каждого типа продукта (`product_type`).

**Решение:**

```sql
SELECT 
    s.sales_transaction_date::DATE as sale_date,
    p.product_type,
    s.sales_amount,
    SUM(s.sales_amount) OVER (
        PARTITION BY p.product_type 
        ORDER BY s.sales_transaction_date
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) as running_total_by_type
FROM sales s
JOIN products p ON s.product_id = p.product_id
ORDER BY p.product_type, s.sales_transaction_date;
```

**Пояснение:**

* `PARTITION BY product_type` — расчёт отдельно для каждого типа товара.
* `ORDER BY sales_transaction_date` — сортировка по дате.
* `ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW` — нарастающий итог от начала до текущей строки.
* `JOIN` — нужен для получения типа продукта.
